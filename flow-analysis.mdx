## Overview

Flow Analysis tests complete user journeys to identify missing paths, states, and friction points. Analyze static flows for edge cases or test interactive prototypes with real user tasks.

**Time per analysis**: ~2-3 minutes  
**Cost**: 75 credits per analysis  
**Frames**: 1-6 for missing states, 2-6 for flow testing  

---

## Two Features

Choose how you want to analyze your flow:

### Missing States

Analyze static screens for missing states and edge cases.

**Select 1-6 frames** that are part of a flow. Floto identifies:
- Missing states and scenarios
- Incomplete edge cases
- Gaps in user journey
- State transitions that are missing

### Flow Testing

Test interactive prototypes to see how users navigate.

**Select 2-6 frames** (interactive prototype) **+ define a task**.

Task example: "Sign up for an account", "Complete checkout", "Find the pricing page"

Floto simulates:
- How users move through your prototype
- Where they get stuck or hesitate
- Where they'd drop off
- Unclear navigation paths

## How It Works

### Step 1: Open Flow Analysis

Click **Flow Analysis** in the Floto plugin.

Test complete user journeys—onboarding, checkout, and any critical path. See where users get stuck before they hit production.

### Step 2: Choose Analysis Type

Select one of two types:

- **Missing States** - Find missing states and incomplete scenarios
- **Flow Testing** - Test how users move through your prototype

### Step 3: Select Frames

**For Missing States:**
- 1-6 frames representing your flow
- Can be screens, wireframes, mockups, or components

**For Flow Testing:**
- 2-6 frames (interactive prototype with navigation)
- Frames should connect in a logical flow

### Step 4: Define Task (Flow Testing Only)

Describe what users should accomplish:
- "Sign up for an account"
- "Complete checkout"
- "Find pricing and select a plan"
- "Reset a forgotten password"

Floto tests how users navigate to achieve this goal.

### Step 5: Run Analysis

Processing starts automatically.

**Missing States** (~5 processing stages):
1. Processing frames
2. Mapping user journeys
3. Identifying missing states
4. Checking edge cases
5. Generating report

**Flow Testing** (~5 processing stages):
1. Analyzing prototype structure
2. Mapping navigation paths
3. Preparing AI agent
4. Initializing flow test
5. Getting ready to start

## Understanding Results

### Missing States Report

For each gap identified:
- **State** - What's missing
- **Context** - Where in the flow it's missing
- **Impact** - Why it matters for user experience
- **Fix** - How to add the missing state

### Flow Test Report

See how users navigate:
- **Path taken** - Which screens the AI visited
- **Task completion** - Did they achieve the goal?
- **Friction points** - Where they hesitated
- **Unclear interactions** - Buttons/paths that confused them
- **Drop-off points** - Where they got stuck

## Frame Requirements

### Supported Frames

- Figma frames, screens, components

### Frame Size

- No minimum size
- Maximum: 20MB per frame
- Recommended: 320px - 1920px width

### Frame Count

**Missing States**: 1-6 frames
**Flow Testing**: 2-6 frames (interactive with navigation)

## FAQ

<Accordion title="Which should I use, Missing States or Flow Testing?">
**Missing States** if you want to find incomplete states and edge cases in static designs. **Flow Testing** if you have an interactive prototype and want to see how users navigate to accomplish a task.
</Accordion>

<Accordion title="Do I need a fully interactive prototype for Flow Testing?">
Yes. Frames should have logical navigation so users can move between them based on the task goal.
</Accordion>

<Accordion title="Can I test multiple flows?">
Yes. Run separate analyses for different user journeys (onboarding, checkout, account settings, etc.).
</Accordion>

<Accordion title="What if my frames aren't interactive?">
Use Missing States instead. It analyzes static screens without requiring navigation.
</Accordion>

<Accordion title="How many credits does an analysis cost?">
Each analysis costs **75 credits**, whether Missing States or Flow Testing.
</Accordion>

<Accordion title="Can I see where exactly users get stuck?">
Yes. Flow Testing shows friction points, hesitations, and drop-off locations in your flow.
</Accordion>

<Accordion title="What if the flow test doesn't complete the task?">
The report shows where the AI got stuck or dropped off. Review that step to fix the unclear navigation or missing interaction.
</Accordion>

<Accordion title="Can I share flow analysis results?">
Save and reference results from your project history. Share through your team workspace.
</Accordion>
